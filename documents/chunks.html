Current state:

- EmbeddingModelConfig has two variants: BgeSmallEnV15 and BgeSmallEnV15Q (quantized) — both 384

dims

- ONNX_MODEL_PATH (env var) already controls which .onnx file loads — the loader is generic

- The critical constraint: SimpleTokenizer in onnx_embedder.rs (line 189) uses seahash to hash

words into token IDs — it's not a real BPE/WordPiece tokenizer

The tokenizer is the real blocker. It maps each whitespace-split word to seahash(word) % 29522 +

1000, which mimics BERT's vocab size but with random IDs. No standard sentence transformer model

(BGE, MiniLM, E5) will produce meaningful embeddings with this input — they expect proper subword

tokenization.

Practical options once tokenizer is fixed:

- all-MiniLM-L6-v2 — 384 dims, drop-in size, widely used, good general quality

- bge-base-en-v1.5 — 768 dims, significantly better retrieval quality, ~2× model size

- e5-small-v2 — 384 dims, instruction-following ("query: …" / "passage: …" prefix style)
