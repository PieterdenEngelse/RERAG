The Problem
Not all LLM models can handle tool calling (the core of Agentic mode):

✅ Support tool calling: qwen2.5, phi3.5, llama3, etc.
❌ NO tool calling: phi, phi2, phi:latest

Agentic mode absolutely requires tool-calling support because it's an LLM-driven loop where the model decides which tools to call.
The "Something" That Had to Change
The Agentic button needs a guard:

Disable the button when the current model lacks tool-calling support
Show a warning message: "Current model (phi:latest) lacks tool-calling support. Switch to phi3.5, qwen2.5, or llama3 for Agentic mode."
Auto-fallback: If user is in Agentic mode and switches to a non-tool-supporting model → automatically fall back to Auto mode
Greyed-out styling when disabled (cursor:not-allowed, reduced opacity, etc.)
