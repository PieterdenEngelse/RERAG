What the threshold controls

The threshold defines how different a new segment’s embedding can be from the current chunk centroid before you declare a topic shift.

  • Higher threshold (0.85–0.95) → stricter → more chunks

  • Lower threshold (0.60–0.75) → looser → fewer chunks

Cosine similarity ranges from 0–1, so this is a semantic distance gate.

How to pick a good threshold (practical method)

A. Start with defaults

Most embedding-based chunkers use:

  • 0.80–0.85 for general text

  • 0.75 for narrative text

  • 0.90 for technical or tightly structured documents

These are good starting points.

B. Tune based on your embedding model

Different models produce different similarity distributions.

Model type

Typical threshold

OpenAI text-embedding-3-large

0.82–0.88

BGE-large / E5-large

0.78–0.85

MiniLM / small models

0.70–0.80

Smaller models → noisier embeddings → lower threshold.



C. Tune based on your retrieval goals

This is the real architectural decision.

If you want tight topical coherence

(e.g., GraphRAG, knowledge graphs, summarization-first retrieval) → 0.85–0.92

If you want longer, more forgiving chunks

(e.g., classic RAG, QA over long docs) → 0.75–0.82

If your text is highly narrative

→ 0.70–0.78

D. Empirical tuning (best method)

  1. Take 20–50 representative documents.

  2. Run chunking with thresholds:

    • 0.70

    • 0.80

    • 0.85

    • 0.90

  3. Inspect chunk boundaries.

  4. Pick the threshold that best matches human intuition.

This is the only reliable method because embedding distributions vary.
